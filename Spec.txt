遊戲規格書：聽覺色盤 - 喵喵調音師 (Sonic Palette: Meow Edition)

1. 遊戲概述 (Overview)

這是一款基於純網頁前端技術的聽覺解謎遊戲。遊戲將測試玩家對聲音細節（音高、音色、沙啞度）的敏銳度。玩家需要聆聽一段隨機生成的「目標貓叫聲」，然後利用三個滑桿（Sliders）將原始的貓叫聲調整至與目標聲音一致。遊戲共進行 5 個回合，最後計算總分。

2. 技術棧 (Tech Stack)

前端結構： HTML5

視覺樣式： CSS3 (極簡暗黑風格 Dark Mode，搭配霓虹強調色 #00FFAA 和 #B266FF)

核心邏輯： Vanilla JavaScript (ES6+)

音訊引擎 (核心)： Web Audio API

AudioContext

AudioBufferSourceNode (用於載入和播放音訊檔，不循環播放)

BiquadFilterNode (用於控制音色 Timbre)

WaveShaperNode (新增：用於控制沙啞度/失真 Grit)

GainNode (用於全域主音量 Master Volume)

3. 核心音訊物理對射 (Audio Mechanics Mapping)

音頻節點連接順序：Source ➔ Filter (Timbre) ➔ WaveShaper (Grit) ➔ Master Gain (Volume) ➔ Destination

基礎聲源 (Base Sound)： 系統預設載入 dragon-studio-cat-meow-401729.mp3（或使用玩家上傳的音檔）。播放時設定 loop = false (播放一次即停)。

變數 1：Pitch (音高)

控制對象： AudioBufferSourceNode.playbackRate.value

數值範圍： 0.5 (最低音/大叔貓) 到 2.0 (最高音/幼貓)

變數 2：Timbre (音色/共鳴)

控制對象： BiquadFilterNode

節點設定： type = 'peaking', Q.value = 5, gain.value = 15

控制參數： BiquadFilterNode.frequency.value

數值範圍： 300 (Hz, 沉悶胸腔共鳴) 到 3000 (Hz, 尖銳鼻腔共鳴)

變數 3：Grit (沙啞度 / 失真)

控制對象： WaveShaperNode.curve

節點設定： 使用 JavaScript 生成一條失真曲線 (Distortion Curve) Float32Array。

控制參數： 曲線的極端程度 (Amount)。

數值範圍： 0 (完全純淨無失真) 到 50 (強烈拆聲/野貓呼嚕聲)

4. 遊戲流程 (Game Flow)

全域 UI 元件 (隨時可用)

Master Volume Bar (主音量滑桿)： 位於畫面角落，控制連接至 Destination 前的 GainNode，預設為 0.5。讓玩家隨時調整整體噪音大小。

狀態 0：初始化 (Init)

畫面顯示一個可選的 File Input，提示玩家「(選填) 上傳自訂貓叫聲」。

若無上傳，系統自動載入內建的 dragon-studio-cat-meow-401729.mp3。

解析成功後，顯示「Start Game」按鈕。

狀態 1：聆聽階段 (Phase: Listen) - 重複 5 圈

系統使用 Math.random() 為 Pitch, Timbre, Echo 隨機生成一組目標數值 (Target Values)。

隱藏/禁用三個玩家控制滑桿。

畫面顯示「Listen to the Target... (正在聆聽目標聲音)」。

應用 Target Values 播放聲音，持續 5 秒鐘。

5 秒後，停止播放目標聲音，自動進入狀態 2。

狀態 2：調音還原階段 (Phase: Tune)

畫面顯示 3 個 <input type="range"> 滑桿 (Pitch, Timbre, Echo)。

滑桿初始位置應隨機放置，或預設在正中間。

重要：不能在介面上顯示具體的 Hz、比例或倍速數字，必須讓玩家純靠聽覺判斷。

玩家拖動滑桿 (oninput 事件) 時，實時 (Real-time) 改變音訊節點的參數，並持續循環播放 (Loop) 玩家目前設定的聲音。

玩家認為調整完畢後，點擊「Confirm (確認)」按鈕。

點擊後，進入狀態 3。

狀態 3：回合結算 (Phase: Round Score)

計算該回合分數。

視覺對比動畫： 在玩家的滑桿上，以霓虹綠色標示出 Target 的正確位置，直觀顯示誤差。

分數揭曉動畫： 顯示該回合準確度百分比 (數字動態遞增至最終結果)。

對比播放機制 (防止嘈雜)： 廢除 Hold/Release 自動發聲設計。改為提供兩個獨立的按鈕 「Play Target」 與 「Play Your Tune」。結算畫面預設為靜音，玩家必須主動點擊才會播放對應聲音 一次，絕不自動循環或重疊播放，保持安靜。

點擊「Next Round」進入下一回合。

狀態 4：遊戲結束 (Phase: Game Over)

顯示 5 個回合的總平均分與評價。

提供「Play Again」按鈕。

5. UI/UX 設計要求 (UI/UX Design)

主音量控制： 需獨立於遊戲謎題之外，清晰標示為系統音量。

按鈕狀態： 點擊 Play 播放聲音時，按鈕應有視覺回饋（例如發光或變色），播放完畢後恢復，避免玩家狂按重疊播放。

6. 計分演算法 (Scoring Algorithm)

計算單項誤差：

Pitch Score = 100 - ( Math.abs(T_pitch - P_pitch) / (2.0 - 0.5) * 100 )

Timbre Score = 100 - ( Math.abs(T_timbre - P_timbre) / (3000 - 300) * 100 )

Grit Score = 100 - ( Math.abs(T_grit - P_grit) / (50 - 0) * 100 )

確保分數不為負數： 若 < 0 則設為 0。

回合總分： (Pitch Score + Timbre Score + Grit Score) / 3 (保留兩位小數)。
